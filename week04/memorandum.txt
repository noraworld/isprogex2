・scanfとfgetsの併用は後々面倒くさい。バッファの調整(処理)が必要。

・fgetsも完璧ではない。オーバーフローのときにバッファに溜め込む。

・ファイルオープンできなかったとき(fp == NULL のとき)はオープンしたことにならない。
　(fp = fopen(...) == NULL)のときfclose(fp)するとセグメンテーション違反となる。

・fgetsの文言や各種の機能に分かれているときは関数を使った方が良い。

・1文字ずつ確認して、その条件に合わないものが複数出るようなときは
　そのループ内にbreakを書いておけば1回だけのエラーメッセージで見通しがすっきりする。

【超重要!!!!】
コンパイルエラーの表示内容について

[例]
option04.c: In function 'main':
option04.c: 33: error: stray '\343' in program
option04.c: 33: error: stray '\200' in program
option04.c: 33: error: stray '\200' in program

このようなエラーが出てきたら
文法が間違っているのではなく
不正な文字入力がされていることを表す

具体的には全角スペースが
誤って入ってしまっている可能性が高い
